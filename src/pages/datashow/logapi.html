日志服务关键字概念说明

  • yylogid: 日志服务中的租户id
  • yylogappid:日志服务中租户下的appid
  • token：验证，与yylogid.yylogappid 确定唯一，不产生变化
  • attribute：字符串数组，保存日志文件分区信息[！attribute确定后，便不建议更改，否则查找不到更改前的日志]

一、日志接收接口

须知：

1 使用前与服务人员约定attribute，content格式，用yylogid,yylogappid申请token

2 接口设计灵活，可接收json格式因用户需求而不同，文档会做举例说明

3 接口做token验证，需传入正确的yylogid、yylogappid、token，

4 attribute中字段在用户传入的json中有对应值

5 接口/UniteLog/web/log/receive中不做任何逻辑验证处理做上传数据，无论用户发送什么都返回success，在开发时，请使用

. 接口/UniteLog/web/log/develop/receive此接口会做各种验证，只有验证通过了，日志最终才能给上传成功

. 线上环境如果单位时间日志数据上传量很大，避免504，选择使用接口/UniteLog/web/log/receive

接口地址：

api:  接口/UniteLog/web/log/develop/receive

日志接收接口json格式：

{
    "token": "",
    "yylogid": "",
    "yylogappid": "",
    "content": ,                                //content格式因用户需求而定，可以是string，map。。。
    "timestamp": "",
    "timestampformat": "",
    "attribute": [
        "yylogid",
        "yylogappid"
        ...                                        //attribute字符串数组中，yylogid与yylogappid是必需的，后面是约定的其他字段，下面会举例说明
    ]
    ...                                            //以上是日志服务的固定字段，这里可能存在用户确定的其他字段，下面会举例说明
}

二、实时日志查询接口

须知：

1 实时日志保存近7天，7天外日志使用历史日志查询或下载日志

2 termQuery中条件参考content{}格式

3 keyword关键字查询不支持模糊查询

4 默认查询1000条日志，最多查询1000条日志

5 yylogid yylogappid fromdate todate dateformat 不为空

接口地址：

api:  域名/UniteLog/searchlog/v1/search/logs

日志接收接口json格式：

{
    "yylogid": "",
    "yylogappid": "",
    "fromdate": "",                    //查询时间段
    "todate": "",
    "dateformat": "",
    "limit": "",
    "keyword": "",
    "termQuery": {

    },
    ...                                //attribute约定的其他字段参数
}

实时查询json举例1： 对应日志上传第一个举例

{        
    "yylogid": "developer-yonyou-com",
    "yylogappid": "developtest",        
    "fromdate": "20180416091720291",
    "todate": "20180521091720291",
    "dateformat": "yyyyMMddHHmmssSSS",
    "limit": "1000",
    "keyword": "aaa",                            //不支持模糊查询，如果是  aa或其他就查询不到
    "termQuery": {
        "dockerid": "49b6d26fd20001230958bcbf5d1cf30f95db11175dde6718d4dba0057df1272e",
        "podname": "fog64200-7d5fcf564c-kgs65"
    },
    "namespace": "c87e2267-1001-4c70-bb2a-ab41f3b81aa3",
    "appid": "fog64200"
}

三、历史日志查询接口

须知：

1 历史日志查询接口json格式逻辑通实时查询

2 并非日志上传，就可以立即查询到历史日志（impala存）

3 keyword支持模糊查询

4 默认查询1000条日志，最多查询1000条日志

5 yylogid yylogappid fromdate todate dateformat 不为空

接口地址：

api:  域名/UniteLog/searchlog/v1/search/historylog

日志接收接口json格式：

{
    "yylogid": "",
    "yylogappid": "",
    "fromdate": "",                    //查询时间段
    "todate": "",
    "dateformat": "",
    "limit": "",
    "keyword": "",
    "termQuery": {

    },
    ...                                //attribute约定的其他字段参数
}

四、日志下载

4.1、日志下载查询接口

须知：

1 日志下载有2接口，一是日志下载查询（post请求）二是日志下载接口（get请求）

2 日志下载查询接口json格式逻辑通实时查询,但没有keyword，termQuery，limit

3 yylogid yylogappid fromdate todate dateformat 不为空

接口地址：

api:  域名/UniteLog/searchlog/v1/searchload/logs

日志接收接口json格式：

{
"yylogid": "developer-yonyou-com",
"yylogappid": "developtest",
"fromdate": "20180416091720291",
"todate": "20180621091720291",
"dateformat": "yyyyMMddHHmmssSSS"，
...                                            //attribute约定的其他字段参数
}

日志下载查询json举例截图：

4.2、日志下载接口

须知：

1 日志下载接口为get请求

2 传参path 即为在日志下载查询接口获取的路径

3 后续迭代会增加日志下载权限校验功能，除path参数还需其他参数传递（待定）

接口地址：

api:  域名/UniteLog/searchlog/v1/download/logs?path=

日志接收接口json格式举例：

http://logservice.app.yyuap.com/UniteLog/searchlog/v1/download/logs?path=developer-yonyou-com/developtest/c87e2267-1001-4c70-bb2a-ab41f3b81aa3/fog64200/2018-05-16/10/stdout.1526453077455.log.tmp
